# 项目 README：多生物圈耦合模拟器 (Multi-Biosphere Coupling Simulator)

## 1. 项目目标

本项目旨在构建一个**可交互、可扩展的封闭生态系统模拟平台**，用户能够：

- **自定义搭建**单个“生物圈”（Biosphere）：自由配置其生物群落（生产者/消费者/分解者的种类与初始数量）、非生物环境（大气成分、水体体积、土壤有机质等）以及物理参数（容积、光照曲线、温度调节能力）。
- **创建多个独立的生物圈**：每个生物圈拥有自己的状态变量和内部动力学。
- **实现物质交换**：在不同生物圈之间定义交换通道（例如通过管道、大气连通或水体流动），交换的物质可包括 CO₂、O₂、水汽、有机碎屑甚至生物个体（迁移）。
- **运行模拟并可视化**：观察每个生物圈的碳/氧/水/生物量随时间演变，以及跨生物圈的物质流。
- **实验与教学**：验证“生物圈2号”的失败原因（如碳失衡、氧衰减），探索多个小生态系统的耦合效应（例如一个圈生产氧气，另一个圈消耗氧气并输出 CO₂）。

**典型应用场景**：
- 研究多个孤立生态系统在有限物质交换下的稳定性。
- 模拟“地球化改造”中多个栖息地之间的资源循环。
- 设计一个太空基地内不同舱段（居住舱、农业舱、藻类舱）之间的气体与水平衡。

## 2. 数学模型选择

为了在**计算效率**与**生态学真实性**之间取得平衡，我们采用**分区室（Compartment）动力学 + 常微分方程（ODE）** 建模方法，并在多生物圈之间通过**耦合通量项**进行扩展。

### 2.1 单个生物圈的内部模型

每个生物圈被视为一个**完全混合的区室**（well-mixed compartment），其状态由以下核心变量描述（可根据用户配置增减）：

| 变量 | 符号 | 单位 | 说明 |
|------|------|------|------|
| 大气 CO₂ 质量 | \(C_a\) | g C | 自由气体 |
| 大气 O₂ 质量 | \(O_a\) | g O₂ | 自由气体 |
| 植物生物量 | \(B_p\) | g C | 生产者 |
| 消费者生物量 | \(B_c\) | g C | 消费者（动物/人） |
| 死有机质 | \(D\) | g C | 凋落物、粪便 |
| 水体溶解无机碳 | \(W\) | g C | 可选，提供缓冲 |
| 温度 | \(T\) | °C | 影响所有代谢速率 |

**关键过程**（以碳循环为例，氧循环通过化学计量系数关联）：
- 光合作用：\( \text{Photo} = P_{\max} \cdot f_{\text{light}}(t) \cdot \frac{C_a}{C_a + K_m} \cdot B_p \)
- 植物呼吸：\( R_p = r_p \cdot B_p \cdot Q_{10}^{(T-20)/10} \)
- 消费者摄食：\( \text{Grazing} = g_{\max} \cdot \frac{B_p}{B_p + K_g} \cdot B_c \)
- 消费者呼吸与粪便：分别进入 CO₂ 和死有机质
- 微生物分解：\( \text{Decomp} = d_{\max} \cdot \frac{D}{D + K_d} \cdot Q_{10}^{(T-20)/10} \)
- 气体交换（水体 ↔ 大气）：\( \text{GasExchange} = k_{\text{gas}} \cdot (C_{\text{eq}} - C_a) \)

每个变量对应一个 ODE，形式为：
\[
\frac{dX_i}{dt} = \sum \text{源过程} - \sum \text{汇过程}
\]

**微分方程组的求解**：采用自适应步长的 **BDF 方法**（`scipy.integrate.solve_ivp` 的 `method='BDF'`），以处理刚性系统（不同过程时间尺度差异可达 10^3 倍）。

### 2.2 多生物圈之间的物质交换

设有 \(N\) 个生物圈，每个生物圈 \(i\) 拥有一组状态变量 \(\mathbf{X}^{(i)}\)。我们允许用户定义**交换矩阵** \(E^{(i \to j)}_{m}\)，表示物质 \(m\) 从生物圈 \(i\) 到 \(j\) 的通量（单位：g/day）。

**一般形式的耦合 ODE**：
\[
\frac{dX_m^{(i)}}{dt} = \underbrace{\text{内部过程}^{(i)}}_{\text{单圈模型}} + \underbrace{\sum_{j \neq i} \left( E^{(j \to i)}_{m} - E^{(i \to j)}_{m} \right)}_{\text{跨圈净通量}}
\]

**支持的交换模式**：

1. **扩散式交换**（如气体通过半透膜）：
   \[
   E^{(i \to j)}_{m} = k_{ij} \cdot \left( \frac{X_m^{(i)}}{V_i} - \frac{X_m^{(j)}}{V_j} \right)
   \]
   其中 \(k_{ij}\) 为交换系数（m³/day），\(V_i\) 为生物圈 \(i\) 的体积。

2. **泵送式交换**（固定方向流量）：
   \[
   E^{(i \to j)}_{m} = Q_{ij} \cdot \frac{X_m^{(i)}}{V_i}
   \]
   其中 \(Q_{ij}\) 为体积流量（m³/day）。

3. **生物介导的迁移**（例如动物从一个圈移动到另一个圈）：
   - 需要额外的迁移率参数，并且迁移量受限于源圈的生物量。

**用户自定义**：通过一个 JSON 或 YAML 文件定义生物圈之间的连接拓扑和每个交换通道的类型与系数。

### 2.3 模型简化与扩展选项

为了适应不同用户需求，我们提供以下**模型切换**：

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| **基础模式** | 仅碳循环（CO₂ + 植物 + 消费者 + 分解者） | 教学、快速原型 |
| **标准模式** | 碳 + 氧循环（化学计量耦合） | 研究呼吸/光合平衡 |
| **完整模式** | 碳 + 氧 + 水循环 + 温度动力学 | 真实系统模拟 |
| **自定义模式** | 用户选择启用哪些物质（如 N₂、CH₄）和过程 | 高级实验 |

### 2.4 数值稳定性与时间尺度

- **时间单位**：天（day）
- **模拟时长**：典型为 1~10 年（365~3650 天）
- **求解器容差**：相对容差 `rtol=1e-6`，绝对容差 `atol=1e-9`（针对质量变量）
- **非负约束**：所有生物量、气体质量强制为非负（通过在导数中限幅或使用 `solve_ivp` 的 `events` 功能）

### 2.5 参数校准依据

- 植物生理参数：参考《植物生理生态学》（Lambers 等）及典型 C3/C4 植物数据。
- 消费者代谢：基于异养生物呼吸商（RQ ≈ 0.8~1.0）和生长效率。
- 分解速率：来自土壤有机质分解模型（如 CENTURY 模型中的参数）。
- 生物圈2号实测数据：用于验证 O₂ 下降和 CO₂ 振荡现象。

## 4. 使用示例（用户自定义配置）

用户创建一个 YAML 文件，例如 `my_design.yaml`：

```yaml
biospheres:
  - name: "Forest"
    volume: 1000.0  # m³
    initial: {Ca: 400, Bp: 500, Bc: 10, D: 50, T: 20}
    params: {Pmax: 0.3, rp: 0.02, gmax: 0.1, ...}
  - name: "Algae_Tank"
    volume: 100.0
    initial: {Ca: 400, Bp: 100, Bc: 0, D: 5, T: 25}
    params: {Pmax: 0.5, rp: 0.01, gmax: 0.0, ...}  # 无消费者
couplings:
  - from: "Forest"
    to: "Algae_Tank"
    substance: "CO2"
    type: "diffusion"
    coefficient: 0.5  # m³/day
  - from: "Algae_Tank"
    to: "Forest"
    substance: "O2"
    type: "pump"
    flow_rate: 0.1   # m³/day
```

运行：
```bash
python src/ui/cli.py --config my_design.yaml --duration 730 --plot
```

输出：每个生物圈的时间序列曲线以及跨圈物质累积交换量。

## 5. 未来扩展计划

- **随机性模块**：加入天气噪声、种群随机死亡。
- **空间一维化**：允许生物圈内部有空间梯度（如垂直分层）。
- **遗传算法优化**：自动搜索用户定义的“稳定生态结构”参数。
- **实时 Web 协作**：多人同时设计各自的生物圈，并让它们相互交换物质。
